Noita Inspired Sand Dropper DEMO

Description: This Project was an assignment for my Intro Into Java class Final this project consisted of making a sand dropper game that had its own physics. For example if placing sand the sand would spread and pile like in the real world, Water would spread and level out, metal would stay put suspended in air manipulating how sand and water would fall. Lastly there was ice that would freeze all water touching it and oil which acted like water but would always float on top of water.

Technologies/Skills: Required use of GUI for the interface as well as basic knowledge of Java as well as how to use your development enviorment. A grasp of object oriented programming was extreamly important to making sure elements worked together properly.

Outcome: This project may have pushed me the hardest sheerly due to the fact that I had basic no knowledge of software development beforehand, I was pushed to learn and adapt so quickly I know I made some mistakes. This porject game me a firm understanding of the basics of java as well as object oriented programming which helped tremdously in projects and classes moving forward. I would love to revist this project and see how I can improve and make work better in the future.

Othello 2-Player/AI Game

Description: This Project was an assignment for my Intermediate Java class and was around the middle of the semester and was two players trying to get 5 of their pieces in a row simlar to connect 4 but on a 2D flat grid with extra limitations. I was also in charge of programming an AI opponent if the player chose that made smart decisions.

Technologies/Skills: Required the use of a GUI to display the game board as well as a strong understanding of the Java fundamentals. I was required to use problem solving skills to code and making an intelligent AI that made moves to win and beat the player.

Outcome: This project was hard for a diffrent reason that the Noita Project as it was easy to impliment the rules of the game and point scoring and background actions. The hard part was developing an AI that made moves that were smart, I wasnt allowed to make an AI that just made the next avalible move as that would be too easy, the way I got around this issue was by assigning a point value to pieces that were in a row therefore allowing the AI to know which pieces would get it closer to winning and if the point value of the players color or the AI's color was larger would determine if the AI would block the players next move or advance their own board state.

Kitty Klicker Idle Game

Description: Lastly this project was my final for Introduction to Game Design and was an Idle, Clicker Game that consisted of clicking a kitty to get currency to buy upgrades that increased your currency more. This would continue until you had enough currency to prestiege and then retart at a faster accelerated pace. All assests were made myself.

Technologies/Skills: This projected required skills in design, conceptuilzing ideas and executing them, grasp on pixel art and required tools, Unity Engine to run my game, as well as learning the C# programming language.

Outcome: Saying the Noita Game was the harderst was a mistake this project by far pushed me the most, I was tasked with learning a game engine, learn a new programming language as well learn to deliver good looking pixel art all for this final project. I was pushed the furtherest in this project than in any of my others. Kitty Klicker was my most succesful project and is my prize possession I worked hard for months to have it come out perfectly and It has been the most I've improved. I dont plan to come back to this, but I do have many plans for other video games in the future with bigger scope than Kitty Klicker.
